Libraries Release Notes

Cocos2d Library
  • Cocos2d version 3.3

After Update Compilation Help

Because of changes in the Cocos2d 3.3 library files & code structure, user action is required to make an old project compile.
A. Remove all LevelHelper 2 demo source code files as they will no longer compile after the Cocos2d 3.3 update.
All demo files were updated to the new Cocos2d 3.3 library but you will need to create a new project.
1. Inside Xcode right click on the "Demo" folder and choose "Remove Reference" or "Move To Trash"
B. Change initial scene and update Mac main.m file.

1. Inside Xcode, locate "Supporting Files" -> MacOS -> main.m and remove this line "[CCGLView load_];"
2. If your project is still loading LHSceneIntroduction as the initial scene, edit AppDelegateMacOS.m and AppDelegate.m and load your own scene file.
C. Change your Xcode project "Header Search Paths"
1. Select you [ProjectName]+Box2d+iOS target
2. Navigate to Build Settings
3. Search for "Header Search Paths"
4. Double click and add the following paths
  • "$(SRCROOT)/cocos2d/cocos2d" - "recursive"
  • "$(SRCROOT)/cocos2d/external/Chipmunk/objectivec/include" - "non-recursive"
  • "$(SRCROOT)/cocos2d/external/Chipmunk/chipmunk/include" - "non-recursive"
  • "$(SRCROOT)/cocos2d/cocos2d-ui" - "recursive"
  • "$(SRCROOT)/cocos2d/external/ObjectAL" - "recursive"
  • "$(SRCROOT)/Box2D" - "recursive"
  • "$(SRCROOT)/LevelHelper2-API/LevelHelper2-API" - "recursive"
5. Remove the old paths, but make sure you keep your own paths if any.
6. In the Genaral tab add "libcocos2d.a" under Linked Frameworks and Libraries.
7. In the "Build Phases" tab add "cocos2d-ios" under Target Dependencies. 8. Clean and build the project.
